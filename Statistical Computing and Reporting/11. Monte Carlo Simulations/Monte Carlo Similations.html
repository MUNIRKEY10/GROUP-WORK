Ordinary Monte Carlo

The Monte Carlo method refers to the theory and practice of learning about probability distributions by simulation rather than calculus. In ordinary Monte Carlo (OMC) we use IID simulations from the distribution of interest. Suppose X1,X2,X1,X2, are IID simulations from some distribution, and suppose we want to know an expectation

θ=E[Y1]=E[g(X1)].θ=E[Y1]=E[g(X1)].
The law of large numbers (LLN) then says
yn=1n∑i=1nYi=1n∑i=1ng(Xi)yn=1n∑i=1nYi=1n∑i=1ng(Xi)
converges in probability to θθ.

The central limit theorem (CLT) says

n−−√(yn−θ)σ→dN(0,1).n(yn−θ)σ→dN(0,1).
That is, for sufficiently large nn,
yn∼N(θ,σ2/n).yn∼N(θ,σ2/n).
Further, we can estimate the standard error σ/n−−√σ/n with sn/n−−√sn/n where snsn is the sample standard deviation.

Ordinary Monte Carlo

The theory of OMC is just the theory of frequentist statistical inference. The only differences are that

  • the data X1,,XnX1,,Xn are computer simulations rather than measurements on objects in the real world
  • the sample size nn is the number of computer simulations rather than the size of some real world data
  • the unknown parameter θθ is in principle completely known, given by some integral, which we are unable to do.

Approximating the Binomial

Suppose we flip a coin 10 times and we want to know the probability of getting more than 3 heads. This is trivial for the Binomial distribution, which we'll ignore.

runs <- 10000
one.trial <- function(){
  sum(sample(c(0,1),10,replace=T)) > 3
}
mc.binom <- sum(replicate(runs,one.trial()))/runs
mc.binom
## [1] 0.8233
pbinom(3,10,0.5,lower.tail=FALSE)
## [1] 0.828125

Aproximating ππ

  • Area of a circle is πr2πr2
  • If we draw a square containing that circle its area will be 4r24r2
  • So the ratio of the area of the circle to the area of the square is
    πr24r2=π4πr24r2=π4
  • Given this fact, we can empirically determine the ratio of the area of the circle to the area of the square we can simply multiply this number by 4 and we'll get our approximation of ππ.
  • How?

Aproximating ππ

  • Randomly sample xx and yy values on the unit square centered at 0
  • If x2+y2≤.52x2+y2≤.52 then the point is in the circle
  • The ratio of points in the circle multiplied by 4 is our estimate of ππ
runs <- 100000
xs <- runif(runs,min=-0.5,max=0.5)
ys <- runif(runs,min=-0.5,max=0.5)
in.circle <- xs^2 + ys^2 <= 0.5^2
mc.pi <- (sum(in.circle)/runs)*4

Aproximating ππ

plot(xs,ys,pch='.',col=ifelse(in.circle,"blue","grey")
     ,xlab='',ylab='',asp=1,
     main=paste("MC Approximation of Pi =",mc.pi))

Example: Sequential stopping rule

eps <- 0.002
len <- diff(interval)
plotting.var <- c(est, interval)
while(len > eps){
    new.x <- rgamma(n, 3/2, scale=1)
    new.y <- 1/((new.x+1)*log(new.x+3))
    y <- cbind(y, new.y)
    est <- mean(y)
    mcse <- sd(y) / sqrt(length(y))
    interval <- est + c(-1,1)*1.96*mcse
    len <- diff(interval)
    plotting.var <- rbind(plotting.var, c(est, interval))
}

Resampling with sample()

Resampling from any existing distribution gives bootstrap estimators

bootstrap.resample <- function (object) sample (object, length(object), replace=TRUE)
replicate(5, bootstrap.resample (6:10))
##      [,1] [,2] [,3] [,4] [,5]

Bootstrap test

The 2-sample t-test checks for differences in means according to a known null distribution. Let's resample and generate the sampling distribution under the bootstrap assumption:

library(MASS)
diff.in.means <- function(df) {
  mean(df[df$Sex=="M","Hwt"]) - mean(df[df$Sex=="F","Hwt"])
}
resample.diffs <- replicate(1000, diff.in.means(cats[bootstrap.resample(1:nrow(cats)),]))

Bootstrap test

hist(resample.diffs); abline(v=diff.in.means(cats), col=2, lwd=3)
